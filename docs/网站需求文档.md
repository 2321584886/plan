# 个人投资追踪网站需求文档

## 1. 项目概述

### 1.1 项目名称
个人投资追踪网站（Personal Investment Tracker）

### 1.2 项目目标
构建一个面向个人使用的投资管理网站，支持用户按周手动录入资产和持仓数据，展示资金增长历史轨迹，并通过图表直观分析当前资产配置。

### 1.3 技术栈
- 前端：React + Vite + JavaScript
- 后端：Node.js + Express
- 数据库：SQLite

---

## 2. 业务范围

### 2.1 包含范围（In Scope）
1. 投资组合管理（新增、查看）
2. 每周快照录入（新增）
3. 每周快照修改（编辑）
4. 自动读取上周快照用于表单预填
5. 历史资产增长折线图
6. 当前持仓占比饼图
7. 清仓资产（数量为 0）保留历史，不出现在当前饼图

### 2.2 不包含范围（Out of Scope）
1. 实时行情或第三方金融数据接入
2. 多用户认证与权限系统（当前为假登录）
3. 自动交易同步
4. 删除快照功能（当前版本不做）

---

## 3. 用户角色与使用场景

### 3.1 用户角色
- 个人投资者（单用户）

### 3.2 核心使用场景
1. 每周末录入本周资产情况
2. 复制上周持仓后只改动变化项，减少重复输入
3. 修正某周录入错误数据
4. 查看资金净值随时间变化
5. 查看当前持仓结构是否过度集中

---

## 4. 功能需求

## 4.1 投资组合管理

### FR-001 获取投资组合列表
- 说明：获取当前用户全部投资组合
- 输入：无
- 输出：投资组合数组（id、name、description、created_at）

### FR-002 新增投资组合
- 说明：新增一个投资组合（如“美股账户”）
- 输入：name（必填）、description（可选）
- 输出：新增后的投资组合对象

## 4.2 每周快照管理

### FR-003 获取最新快照（用于预填）
- 说明：按组合获取最新一期快照和其持仓
- 输入：portfolio_id
- 输出：最新快照对象 + holdings 列表；若不存在返回 null

### FR-004 新增快照
- 说明：新增某周快照，系统计算并保存总资产
- 输入：portfolio_id、snapshot_date、notes、holdings[]
- 规则：同组合同日期不可重复
- 输出：新增快照摘要（id、snapshot_date、total_value）

### FR-005 修改快照
- 说明：修改历史某一期快照及持仓
- 输入：snapshot_id、snapshot_date、notes、holdings[]
- 处理：先清空旧持仓，再写入新持仓并重算 total_value
- 输出：更新后的快照摘要

### FR-006 获取单期快照详情
- 说明：按快照 id 查询快照头与明细
- 输出：快照对象 + holdings[]

## 4.3 图表分析

### FR-007 历史增长曲线
- 说明：返回时间顺序的总资产序列
- 输入：portfolio_id
- 输出：[{ date, total_value }]

### FR-008 当前持仓饼图
- 说明：返回最新快照的有效持仓
- 过滤规则：quantity > 0 且 total_value > 0
- 输出：snapshot_date + data[]

---

## 5. 非功能需求

### NFR-001 性能
- 单次请求响应时间目标：< 500ms（本地环境）

### NFR-002 可用性
- 表单输入清晰、最少操作完成每周更新

### NFR-003 数据一致性
- 快照与持仓写入采用事务，避免部分成功

### NFR-004 可维护性
- 后端路由分模块（portfolios/snapshots/charts）
- 命名规范统一：数据库 snake_case，接口 JSON 建议 camelCase

---

## 6. 数据库需求（SQLite）

### 6.1 users
- id（主键）
- username（唯一）
- created_at

### 6.2 portfolios
- id（主键）
- user_id（外键）
- name
- description
- created_at

### 6.3 weekly_snapshots
- id（主键）
- portfolio_id（外键）
- snapshot_date
- total_value
- notes
- created_at
- 唯一约束：portfolio_id + snapshot_date

### 6.4 snapshot_holdings
- id（主键）
- snapshot_id（外键）
- asset_symbol
- asset_name
- quantity
- price
- total_value

---

## 7. 接口需求（RESTful）

### 7.1 投资组合
- GET /api/portfolios
- POST /api/portfolios
- GET /api/portfolios/:id/snapshots/latest
- GET /api/portfolios/:id/snapshots

### 7.2 快照
- POST /api/snapshots
- PUT /api/snapshots/:id
- GET /api/snapshots/:id

### 7.3 图表
- GET /api/charts/portfolios/:portfolioId/historical-growth
- GET /api/charts/portfolios/:portfolioId/current-holdings

---

## 8. 前端页面需求

### 8.1 Dashboard 页面
- 展示历史增长折线图
- 展示当前持仓占比饼图
- 支持切换投资组合（后续可扩展）

### 8.2 SnapshotEntry 页面
- 快照日期、备注输入
- 持仓行动态新增
- 一键清仓（quantity 置 0）
- 提交保存
- 首次打开自动拉取上周数据预填

---

## 9. 业务规则

1. 资产总值计算：$total\_value = quantity \times price$
2. 快照总值计算：$snapshot.total\_value = \sum holding.total\_value$
3. 清仓资产保留在历史快照中，但不参与当前饼图
4. 同组合同日期最多一条快照

---

## 10. 验收标准（MVP）

1. 能创建投资组合
2. 能新增每周快照并保存持仓
3. 能修改任意已存在快照
4. 新建快照时可自动预填最近一期持仓
5. Dashboard 可正常显示历史折线图
6. Dashboard 可正常显示当前持仓饼图（过滤清仓资产）
7. 接口异常时有明确错误提示（如重复日期）
